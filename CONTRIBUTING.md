# Contributing to Agent Chat

Thanks for your interest in improving Agent Chat! This document explains how to propose
changes and what we expect from contributions.

## Ground rules

- **Language:** all code, comments, documentation, commit messages, issues, and pull
  requests are written in **English**.
- **Code of Conduct:** by participating you agree to our [Code of Conduct](CODE_OF_CONDUCT.md).
- **Governance:** decision-making is described in [GOVERNANCE.md](GOVERNANCE.md).

## Development setup

See [docs/development.md](docs/development.md) for the full guide. In short:

```bash
npm install
npm run dev      # build in watch mode
npm run lint     # lint
npm test         # unit tests
npm run build    # type-check + production build
```

## Coding standards

- **Architecture:** follow the layering in [docs/architecture.md](docs/architecture.md)
  (domain → application → infrastructure → presentation). Dependencies point inward.
- **File size:** keep files at or under ~200 lines. This is enforced by ESLint.
- **Style:** OOP and clean-code principles. Format with `npm run format` (Prettier).
- **Tests:** add unit tests for domain and application logic.

## Commits and pull requests

- Use [Conventional Commits](https://www.conventionalcommits.org/): `feat:`, `fix:`,
  `docs:`, `refactor:`, `test:`, `chore:`, etc.
- Do **not** add co-authorship trailers generated by AI tools.
- Reference any related issue in the PR description and fill in the PR template.
- Keep pull requests focused; one logical change per PR.

## Specifications

Non-trivial features start with a spec in [docs/specs/](docs/specs/). If your change alters
behavior described by a spec, update that spec in the same PR.

## Reporting bugs and requesting features

Use the [issue templates](https://github.com/NicolasArthurDev/obsidian-agent-chat/issues/new/choose).
For security issues, follow [SECURITY.md](SECURITY.md) instead of opening a public issue.
