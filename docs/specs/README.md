# Specifications

Agent Chat is developed spec-first: non-trivial features begin with a specification here,
reviewed before implementation. Specs describe **what** and **why**; the code is the
**how**. Keep specs in sync with behavior — update them in the same pull request that
changes behavior.

## Status legend

- **Draft** — under discussion, not yet agreed.
- **Accepted** — agreed, ready to implement or implemented.
- **Superseded** — replaced by a newer spec.
